# Trading Analysis Lab

Python-based tools for technical analysis, rule-based stock screening, RSI threshold optimization, and simple signal generation on BIST stocks.

This repository contains early-stage algorithmic trading research tools developed for educational and engineering practice purposes. The main goal is not to provide financial advice, but to study systematic market analysis, technical indicators, stock screening logic, and risk-aware decision support using Python.

## Features

* BIST100 stock screening dashboard with a Tkinter GUI
* RSI buy/sell threshold optimization with heatmap output
* Breakout signal scanner using SMA200, previous 20-day high, and previous 10-day low rules
* Timestamped Excel and image outputs
* BIST100 company-name-to-ticker mapping using a CSV database
* Clean project structure with separated source code, input data, and generated outputs

## Tools

### 1. BIST100 Stock Scanner GUI

`bist_stock_scanner_gui.py` is a Tkinter-based dashboard that scans BIST100 stocks and ranks them using a rule-based scoring model.

The scanner evaluates each stock using the following factors:

| Factor            | Meaning                                            |
| ----------------- | -------------------------------------------------- |
| Trend             | Price position relative to SMA50 and SMA200        |
| Momentum          | 3-month price performance                          |
| Relative Strength | 3-month performance compared with BIST100          |
| Valuation         | Trailing P/E ratio                                 |
| Profitability     | Return on equity                                   |
| Liquidity         | Trading volume                                     |
| Volume Activity   | Current volume compared with recent average volume |
| Risk              | Daily volatility penalty                           |

Run:

```bash
python src/bist_stock_scanner_gui.py
```

Output:

* Interactive GUI table
* Timestamped Excel report in the `outputs/` folder

Example generated file:

```text
outputs/bist100_stock_scan_2026-06-10_18-45-12.xlsx
```

---

### 2. RSI Threshold Optimizer

`rsi_threshold_optimizer.py` tests different RSI buy and sell threshold combinations on a selected BIST100 stock.

The user can enter either a BIST100 ticker or a company name from the CSV database.

Example inputs:

```text
Aselsan
THYAO
Türk Hava Yolları
```

Strategy logic:

* Buy when RSI falls below the selected buy threshold
* Sell when RSI rises above the selected sell threshold
* If a position is still open at the end of the test period, it is closed at the latest available price

The optimizer creates a heatmap where each cell represents the net profit generated by one RSI buy/sell threshold pair. The blue star marks the threshold pair that produced the highest final capital in the historical backtest.

Run:

```bash
python src/rsi_threshold_optimizer.py
```

Output:

* Terminal summary report
* Timestamped heatmap image in the `outputs/` folder

Example generated file:

```text
outputs/rsi_optimization_THYAO_2026-06-10_16-25-30.png
```

Important note:

The blue star on the heatmap does not mean “buy now.” It only shows which RSI threshold pair performed best on historical data for the selected stock.

### 3. Breakout Signal Scanner

`breakout_signal_scanner.py` scans selected BIST stocks using a simple breakout-based signal model.

Strategy rules:

| Signal | Rule                                                                  |
| ------ | --------------------------------------------------------------------- |
| BUY    | Close price is above SMA200 and breaks above the previous 20-day high |
| HOLD   | Close price remains above SMA200, but no new breakout occurs          |
| SELL   | Close price falls below the previous 10-day low                       |
| AVOID  | Close price is below SMA200                                           |

Run:

```bash
python src/breakout_signal_scanner.py
```

Output:

* Clean terminal summary
* Timestamped Excel report in the `outputs/` folder

Example generated file:

```text
outputs/breakout_signals_2026-06-10_16-03-41.xlsx
```

## Methodology Notes

This repository uses simple rule-based models. The tools are designed for learning, experimentation, and demonstrating Python-based financial data analysis workflows.

Current limitations:

* Transaction costs are not included
* Slippage is not included
* Position sizing is simplified
* Backtesting logic is basic
* No live trading execution is implemented
* Historical performance does not guarantee future results

Possible future improvements:

* Add transaction cost and slippage modeling
* Add maximum drawdown calculation
* Add win rate and trade count statistics
* Add benchmark comparison
* Improve scoring model transparency
* Add more advanced backtesting reports

## Disclaimer

This project is for educational and research purposes only.

It does not provide financial advice, investment recommendations, trading signals, or portfolio management services. Any analysis produced by these scripts should be interpreted as experimental and should not be used as the sole basis for financial decisions.
